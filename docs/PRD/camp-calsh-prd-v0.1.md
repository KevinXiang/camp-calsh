# 2. 游戏概述

## 2.1 游戏名称

暂定名称：

**《军营大作战》**

备选名称：

* 《红蓝兵营大作战》
* 《无限军营：红蓝战场》
* 《玩具兵团沙盒》
* 《红蓝小兵战争模拟器》
* 《军营沙盘》

---

## 2.2 游戏一句话描述

《军营大作战》是一款运行在 Web 端的 2.5D 放置对战沙盒模拟器。玩家可以在无限画布上自由放置红方和蓝方军营，军营会持续生产 Q 版玩具小兵，小兵自动寻找敌人并使用近战或远程武器进行低暴力对战。玩家通过调整军营数量、位置、兵种和升级配置，观察红蓝双方在沙盒战场中的自动对抗结果。

---

## 2.3 游戏核心定位

本游戏不是传统 RTS，也不是塔防闯关游戏，而是一个偏模拟实验性质的沙盒对战游戏。

玩家身份不是红方或蓝方指挥官，而是：

**战场设计者 / 沙盒观察者 / 对战实验者**

玩家可以同时控制红蓝双方，自由摆放双方军营，创造不同战场局面，并观察自动战斗结果。

# 3. 目标用户与体验目标

## 3.1 目标用户

| 用户类型      | 说明                 |
| --------- | ------------------ |
| 休闲玩家      | 喜欢简单、直观、可观察的自动战斗游戏 |
| 沙盒模拟爱好者   | 喜欢自由摆放单位、观察结果、反复实验 |
| 儿童与亲子用户   | Q版低暴力风格，适合轻松娱乐     |
| 策略模拟玩家    | 喜欢研究兵种搭配、站位、数量对比   |
| Web 小游戏用户 | 希望打开浏览器即可游玩        |

---

## 3.2 核心体验目标

游戏需要提供以下体验：

1. **放置简单**
   玩家可以快速选择红方或蓝方军营，并直接放到战场上。

2. **战斗自动发生**
   军营自动生产小兵，小兵自动移动、寻敌、攻击、死亡。

3. **结果可观察**
   玩家能直观看到哪一方占优，哪种军营组合更强，兵线如何推进。

4. **战局可调整**
   玩家可以暂停、加速、删除、升级、清场，方便反复实验。

5. **画面热闹但不血腥**
   使用 Q版玩具士兵、星星、弹跳、积木碎片等表现战斗，避免血腥和真实战争压迫感。

6. **支持大地图自由扩展**
   玩家可以在无限画布中不断扩大战场规模，形成多个局部战线。

# 4. 游戏核心玩法

## 4.1 核心玩法循环

```text
选择阵营
  ↓
选择军营类型
  ↓
在无限画布中放置军营
  ↓
军营自动生产小兵
  ↓
小兵自动寻找敌人并战斗
  ↓
玩家观察战局变化
  ↓
暂停 / 加速 / 升级 / 删除 / 新增军营
  ↓
继续模拟或重置战场
```

---

## 4.2 玩家主要操作

玩家在游戏中主要进行以下操作：

| 操作   | 说明                  |
| ---- | ------------------- |
| 选择阵营 | 选择放置红方或蓝方军营         |
| 选择军营 | 选择剑兵营、盾兵营、弓兵营、投矛营   |
| 放置军营 | 在地图上点击或触控放置军营       |
| 观察战斗 | 拖动画布、缩放画布，查看不同区域战斗  |
| 暂停模拟 | 暂停战斗，方便布局和查看状态      |
| 加速模拟 | 提高模拟速度，快速观察结果       |
| 升级军营 | 提升产兵速度、小兵血量、小兵攻击    |
| 删除对象 | 删除选中的军营或小兵          |
| 清除小兵 | 保留军营，仅清空当前小兵        |
| 清空战场 | 删除所有军营和小兵，重新开始实验    |
| 查看统计 | 查看双方兵力、军营数、击杀数、战损比等 |

# 5. 游戏模式

## 5.1 MVP 模式：沙盒战场模式

MVP 阶段只实现一个核心模式：

**沙盒战场模式**

在该模式中：

* 玩家可以同时放置红方和蓝方军营。
* 双方军营规则完全相同，仅颜色和敌我关系不同。
* 游戏不设置关卡目标。
* 游戏不强制判定胜负。
* 玩家可以自由实验不同军营组合。
* 玩家可以随时暂停、继续、加速、清场、重置。

---

## 5.2 后续可扩展模式

MVP 之后可以扩展以下模式：

| 模式       | 说明                     |
| -------- | ---------------------- |
| 关卡挑战模式   | 系统给定初始敌方军营，玩家尝试用有限配置获胜 |
| 无限战争模式   | 双方持续扩张，战场不断扩大          |
| 兵种实验室    | 专门测试不同兵种、等级、数量的对抗结果    |
| 防守模拟模式   | 一方防守，一方不断进攻            |
| 自动生成战场模式 | 系统随机生成红蓝双方军营布局         |

---

# 6. 世界与地图设计

## 6.1 地图形式

游戏地图采用：

**2.5D 无限画布战场**

表现方式为：

* 俯视 2D 逻辑地图
* 伪 2.5D 美术表现
* 建筑带有立体感和阴影
* 小兵带有脚底阴影
* 地面可以无限拖拽扩展
* 玩家可自由缩放视角

---

## 6.2 无限画布设计原则

游戏中的“无限画布”不要求数学意义上的真正无限，而是用户体验上的大范围自由地图。

设计原则：

1. 玩家可以持续向四周拖动画布。
2. 玩家可以在较大区域内自由放置军营。
3. 地图不需要明显边界。
4. 当前视野外区域仍然可以继续模拟战斗。
5. 大规模单位数量过多时，需要进行性能优化。
6. 不同缩放级别下，单位显示细节可以不同。

---

## 6.3 地图视觉元素

MVP 阶段地图以简洁为主：

| 元素     | 说明              |
| ------ | --------------- |
| 草地地面   | 默认战斗区域          |
| 网格辅助线  | 可选，用于帮助摆放军营     |
| 阴影     | 增强 2.5D 感       |
| 装饰物    | 石头、木箱、小旗子，可后续加入 |
| 战斗区域提示 | 可选，用于突出前线区域     |

MVP 阶段暂不加入复杂地形阻挡，避免增加寻路复杂度。

# 7. 阵营系统

## 7.1 阵营类型

游戏中有两个阵营：

| 阵营 | 颜色 | 说明    |
| -- | -- | ----- |
| 红方 | 红色 | 玩家可放置 |
| 蓝方 | 蓝色 | 玩家可放置 |

---

## 7.2 阵营规则

1. 红方单位与蓝方单位互为敌人。
2. 同阵营单位之间不会互相攻击。
3. 红蓝双方使用相同的军营类型和小兵规则。
4. 红蓝双方的数值完全一致，保证沙盒对比公平。
5. 阵营差异主要通过颜色、旗帜、血条边框、攻击特效进行区分。

---

# 8. 军营系统

## 8.1 军营定位

军营是游戏的核心建筑。

军营负责：

* 生产小兵
* 决定小兵类型
* 承载升级系统
* 形成兵线来源
* 被敌方小兵攻击和摧毁

---

## 8.2 军营基础规则

| 规则     | 说明                |

| 产兵间隔   | 根据军营类型决定          |
| 是否可升级  | 是                 |
| 是否可删除  | 是，玩家可以主动删除        |

# 9. 小兵系统

## 9.1 小兵基础规则

小兵由军营自动生产，玩家不能直接控制单个小兵。

小兵会自动执行以下行为：

1. 从所属军营出生。
2. 寻找最近敌方目标。
3. 向目标移动。
4. 进入攻击范围后攻击。
5. 目标死亡后重新寻找目标。
6. 自身生命值归零后死亡并消失。

---

## 9.2 小兵通用属性

| 属性   | 说明           |
| ---- | ------------ |
| 阵营   | 红方或蓝方        |

| 攻击力  | 每次攻击造成的伤害    |
| 攻击距离 | 区分近战和远程      |
| 攻击间隔 | 控制攻击频率       |
| 移动速度 | 控制移动快慢       |
| 武器类型 | 近战或远程        |
| 当前目标 | 当前正在追击或攻击的敌人 |
| 所属军营 | 由哪个军营生产      |
| 血条   | 显示当前生命状态     |

# 10. 战斗系统

## 10.1 战斗核心逻辑

小兵战斗逻辑如下：

```text
小兵出生
  ↓
寻找最近敌方小兵
  ↓
如果敌人在攻击范围内，则攻击
  ↓
如果敌人不在攻击范围内，则移动靠近
  ↓
敌方小兵死亡后，重新寻敌
  ↓
如果附近没有敌方小兵，则寻找并攻击敌方军营
  ↓
自身生命值归零后死亡
```

---

## 10.2 目标优先级

小兵选择目标时，按照以下优先级：

| 优先级 | 目标          |
| --: | ----------- |
|   1 | 攻击范围内的敌方小兵  |
|   2 | 最近的敌方小兵     |
|   3 | 最近的敌方军营     |
|   4 | 其他敌方建筑，后续扩展 |

---

## 10.3 近战攻击规则

近战单位包括：

* 剑兵
* 盾兵

近战攻击规则：

1. 必须移动到目标附近才能攻击。
2. 攻击距离较短。
3. 攻击时播放挥砍动画。
4. 命中时目标闪白或弹跳。
5. 使用星星、碰撞光效表现打击感。

---

## 10.4 远程攻击规则

远程单位包括：

* 弓箭手
* 投矛兵

远程攻击规则：

1. 可在较远距离攻击。
2. 攻击时生成远程弹道。
3. 弹道命中目标后造成伤害。
4. 远程单位应尽量保持距离，避免主动冲到近战范围。
5. 弹道表现为玩具箭、软头矛、泡泡弹等低暴力元素。

---

## 10.5 军营攻击规则

当小兵附近没有敌方小兵时，可以攻击敌方军营。

规则：

1. 小兵优先攻击敌方小兵。
2. 没有敌方小兵时，寻找最近敌方军营。
3. 军营生命值归零后被摧毁。
4. 军营被摧毁后停止产兵。
5. 军营被摧毁时，所属仍存活小兵可以继续战斗，直到死亡。

---

## 10.6 死亡与消失表现

由于采用低暴力表现，小兵死亡不使用血腥动画。

推荐表现：

| 事件   | 表现                 |
| ---- | ------------------ |
| 小兵死亡 | 原地晕倒、冒星星、变成玩具零件、淡出 |
| 远程命中 | 小弹花、星星、轻微弹跳        |
| 近战命中 | 玩具碰撞音效、星星闪光        |
| 军营摧毁 | 建筑晃动、积木散落、变成废墟     |
| 单位消失 | 淡出或缩小消失            |

---

# 11. 血条系统

## 11.1 小兵血条

每个小兵头顶显示小型血条。

血条规则：

| 状态   | 表现     |
| ---- | ------ |
| 满血   | 绿色     |
| 中等血量 | 黄色     |
| 低血量  | 红色     |
| 受击时  | 血条短暂闪烁 |
| 死亡时  | 血条消失   |

---

## 11.2 军营血条

军营在以下情况下显示血条：

1. 被玩家选中时。
2. 正在受到攻击时。
3. 鼠标或触控悬停时，电脑端可用。
4. 长按或点击军营时，平板端可用。

---

## 11.3 大规模战斗下的血条优化

当单位数量较多时，需要避免画面混乱。

建议规则：

| 场景    | 血条显示策略           |
| ----- | ---------------- |
| 近距离视角 | 显示完整血条           |
| 中距离视角 | 显示简化血条           |
| 远距离视角 | 隐藏普通小兵血条，仅显示军营血条 |
| 选中单位  | 始终显示血条           |
| 低血单位  | 可突出显示            |

# 12. 升级系统

## 12.1 升级定位

MVP 阶段升级系统用于让玩家快速测试不同强度的军营配置。

因为游戏定位是沙盒模拟器，所以 MVP 阶段不设置复杂资源消耗。

玩家可以直接升级任意军营。

---

## 12.2 军营升级方向

每个军营提供 3 类升级：

| 升级类型 | 效果                 |
| ---- | ------------------ |
| 产兵升级 | 降低产兵间隔，提高出兵速度      |
| 生命升级 | 提升军营生命值和该军营产出小兵生命值 |
| 武器升级 | 提升该军营产出小兵攻击力       |

---

## 12.3 升级规则

1. 每个军营独立升级。
2. 升级只影响该军营以及该军营后续生产的小兵。
3. 已经生产出来的小兵是否同步获得升级，MVP 阶段建议同步更新，方便理解。
4. 每类升级有等级上限。
5. MVP 阶段每类升级最高 3 级。
6. 升级后军营信息面板立即更新数值。
7. 升级后可以有简单视觉反馈，例如军营闪光、旗帜变亮。

---

## 12.4 升级数值建议

### 产兵升级

|   等级 | 效果         |
| ---: | ---------- |
| Lv.1 | 默认产兵速度     |
| Lv.2 | 产兵间隔减少 15% |
| Lv.3 | 产兵间隔减少 30% |

### 生命升级

|   等级 | 效果            |
| ---: | ------------- |
| Lv.1 | 默认生命值         |
| Lv.2 | 军营和小兵生命值 +15% |
| Lv.3 | 军营和小兵生命值 +30% |

### 武器升级

|   等级 | 效果         |
| ---: | ---------- |
| Lv.1 | 默认攻击力      |
| Lv.2 | 小兵攻击力 +15% |
| Lv.3 | 小兵攻击力 +30% |

---

## 12.5 后续装备升级扩展

后续版本可以把数值升级扩展成装备科技。

例如：

|   等级 | 剑兵营表现 |
| ---: | ----- |
| Lv.1 | 木剑兵   |
| Lv.2 | 铁剑兵   |
| Lv.3 | 盾剑兵   |
| Lv.4 | 精锐剑兵  |

|   等级 | 弓兵营表现 |
| ---: | ----- |
| Lv.1 | 木弓手   |
| Lv.2 | 强弓手   |
| Lv.3 | 弩手    |
| Lv.4 | 精锐弩手  |

MVP 阶段只要求数值升级，不强制要求完整装备外观变化。

# 13. 沙盒控制系统

## 13.1 模拟控制

游戏需要提供模拟控制工具。

| 功能    | 说明             |
| ----- | -------------- |
| 暂停    | 停止所有单位移动、攻击、产兵 |
| 播放    | 恢复模拟           |
| 1x 速度 | 正常速度           |
| 2x 速度 | 双倍速度           |
| 4x 速度 | 四倍速度，主要用于快速观察  |
| 清除小兵  | 删除当前所有小兵，保留军营  |
| 清空战场  | 删除所有小兵和军营      |

## 13.2 时间规则

3. 玩家点击播放后开始生产小兵和模拟战斗。
4. 暂停状态下允许放置、删除、升级军营。
5. 播放状态下也允许放置、删除、升级军营。
6. 加速模式下 UI 操作仍然可用。

# 16. 视觉风格需求

## 16.1 总体美术方向

游戏采用：

**Q版玩具士兵 + 低暴力 + 明亮轻松 + 伪 2.5D**

关键词：

* 玩具
* 积木
* 小兵
* 沙盘
* 彩色阵营
* 轻松战斗
* 非真实战争
* 非血腥

---

## 16.2 角色风格

小兵应表现为玩具士兵，而非真实军人。

要求：

1. 身体比例 Q 版。
2. 头大身小。
3. 动作夸张可爱。
4. 武器像玩具，不强调真实杀伤。
5. 红蓝双方通过颜色区分。
6. 不出现血液、残肢、真实伤口。

---

## 16.3 军营风格

军营建议表现为玩具积木建筑。

特点：

1. 有明显红蓝旗帜。
2. 形状简洁，易识别。
3. 不同军营类型外观不同。
4. 建筑带阴影和轻微立体感。
5. 被摧毁后变成积木废墟。

---

## 16.4 战斗特效风格

| 行为   | 推荐表现        |
| ---- | ----------- |
| 近战攻击 | 星星碰撞、玩具敲击   |
| 远程攻击 | 软头箭、泡泡弹、橡胶矛 |
| 受击   | 闪白、轻微弹跳     |
| 死亡   | 晕倒、冒星星、淡出   |
| 军营受击 | 轻微晃动、掉积木    |
| 军营摧毁 | 积木散落、烟圈、废墟  |

# 19. 性能需求

## 19.1 性能目标

游戏运行在 Web 端，需要兼顾电脑浏览器和平板浏览器。

MVP 建议目标：

| 场景      | 目标             |
| ------- | -------------- |
| 普通电脑浏览器 | 300 个小兵流畅运行    |
| 中等平板浏览器 | 150 个小兵基本流畅    |
| 小规模战斗   | 稳定 60 FPS      |
| 中等规模战斗  | 尽量保持 30 FPS 以上 |
| 大规模战斗   | 允许降级显示，但不能卡死   |

---

## 19.2 单位数量限制

MVP 阶段建议设置软限制：

| 对象       |        限制 |
| -------- | --------: |
| 单军营小兵上限  |        20 |
| 推荐最大军营数  |   20 - 30 |
| 推荐总小兵数   | 300 - 500 |
| 平板推荐总小兵数 | 150 - 250 |

当数量过大时，可提示：

```text
当前单位数量较多，可能影响性能。
```

---

## 19.3 性能优化方向

需求层面需要考虑：

1. 远距离单位简化渲染。
2. 视野外单位降低更新频率。
3. 血条按缩放级别显示或隐藏。
4. 弹道和特效数量做上限控制。
5. 小兵寻敌不要每帧全量扫描。
6. 可加入最大单位数量限制。
7. 加速模式下可降低部分动画精度。

# 24. 关键设计风险

## 24.1 单位数量过多导致性能下降

风险：

* 小兵数量持续增长后，浏览器可能卡顿。
* 平板设备性能更敏感。

应对：

* 单军营小兵上限。
* 总单位数量软限制。
* 视野外降频更新。
* 血条和特效按需显示。
* 寻敌逻辑做空间分区优化。

---

## 24.2 战斗画面混乱

风险：

* 小兵太多时，血条、弹道、特效遮挡严重。
* 玩家看不清哪一方占优。

应对：

* 缩放级别控制血条显示。
* 控制特效密度。
* 远距离时用阵营色块或简化单位表现。
* 顶部统计栏提供清晰结果判断。

---

## 24.3 小兵 AI 行为不自然

风险：

* 小兵乱跑。
* 远程兵冲到前排。
* 单位卡住。
* 前线不明显。

应对：

* 优先攻击附近敌方小兵。
* 远程单位保持攻击距离。
* 加入简单分离逻辑。
* 小兵默认向敌方军营方向推进。
* 避免复杂寻路，先用开放地形。

---

## 24.4 沙盒缺少目标感

风险：

* 玩家摆放几次后不知道还能玩什么。

应对：

* 提供统计数据。
* 后续加入战斗报告。
* 后续加入挑战模板。
* 后续加入随机战场。
* 后续加入保存和分享战场配置。

---

# 26. MVP 最终体验目标

MVP 完成后，玩家应该可以完成以下典型玩法：

## 示例玩法 1：基础对抗实验

1. 放置 1 个红方剑兵营。
2. 放置 1 个蓝方剑兵营。
3. 点击播放。
4. 观察双方剑兵交战。
5. 查看统计结果。

---

## 示例玩法 2：近战 vs 远程实验

1. 放置 3 个红方剑兵营。
2. 放置 2 个蓝方弓兵营。
3. 点击播放。
4. 观察近战兵能否冲破远程火力。
5. 调整距离后重新实验。

---

## 示例玩法 3：升级对比实验

1. 放置 1 个红方剑兵营。
2. 放置 1 个蓝方剑兵营。
3. 将红方剑兵营升级武器。
4. 点击播放。
5. 观察升级对战斗结果的影响。

---

## 示例玩法 4：大规模混战实验

1. 放置多个红方军营。
2. 放置多个蓝方军营。
3. 使用不同军营类型组合。
4. 开启 2x 或 4x 加速。
5. 观察战线变化和最终优势方。

---

# 27. 总结

《军营大作战》当前版本的核心目标是：

> 做出一个简单、直观、可扩展的 Web 端 2.5D 红蓝军营放置对战沙盒模拟器。

MVP 阶段不追求复杂关卡、剧情、经济系统或大量兵种，而是优先验证以下核心乐趣：

1. 放置军营是否简单有趣。
2. 自动产兵和自动战斗是否有观赏性。
3. 红蓝双方自由摆放是否能产生有趣战局。
4. 近战和远程组合是否有策略差异。
5. 军营升级是否能明显影响战斗结果。
6. 无限画布是否能带来沙盒扩展感。

只要 MVP 能让玩家反复尝试：

```text
“如果我这样摆，会发生什么？”
```

这款游戏的核心体验就成立了。
