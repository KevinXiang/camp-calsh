# 《军营大作战》UI 与战斗特效优化 PRD

## 1. 文档目标

本文档用于定义当前版本《军营大作战》在以下方向上的体验优化目标：

- 军营 UI
- 士兵可读性
- 武器与投射物表现
- 命中与受击反馈
- 战斗特效层级
- 战场整体信息密度

本文档不讨论新的兵种设计，也不讨论核心战斗规则重写。重点是：

- 让玩家更容易看懂战局
- 让不同军营和士兵更容易被识别
- 让战斗反馈更有节奏和记忆点
- 在不显著增加复杂度的前提下提升整体观感

## 2. 背景与当前问题

当前版本已经完成了基本可玩的 UI 与战斗表现：

- DOM UI 与 Phaser 战斗层已经分离
- 军营外观已经有明显分类
- 单位具备基础动作与武器差异
- 投射物已有直线与抛物线表现
- 命中、爆炸、治疗、盾挡等已有基础特效

这说明系统层面已经具备继续打磨表现的条件。

但从玩家体验上看，当前还有 5 个明显问题。

### 2.1 视觉语言尚未完全统一

当前整体美术方向是：

- 玩具战场
- 积木军营
- Q 版火柴人士兵

这个方向是成立的，但 UI 与特效层有一些元素风格还不够统一，例如：

- DOM UI 仍偏通用网页控件风格
- 命中特效中使用了较多“文字/符号型”表现
- 爆炸、治疗、击中星花之间的语言体系略杂

结果是：

- 游戏“能看懂”
- 但整体气质还不够稳定和鲜明

### 2.2 大规模战斗时单位辨识度下降很快

当前军营辨识度比单位更强。军营已经有比较明确的外观差异，但小兵在混战时的区分主要依赖：

- 阵营色
- 武器形状
- 少量身体姿态差异

当战场单位数量增多后，玩家很容易看见一坨红蓝小人，却难以快速判断：

- 哪一类兵种在前排
- 哪些是高威胁后排
- 哪些是支援或特殊单位

### 2.3 战斗有反馈，但反馈节奏还不够“有层级”

当前已经有：

- 近战命中
- 箭命中
- 投矛命中
- 盾兵格挡
- 爆炸
- 治疗命中
- 营地摧毁

但玩家在体感上还不一定能稳定感受到：

- 轻攻击和重攻击的差别
- 普通远程和关键技能的差别
- 爆破营与火炮营的爆炸不是同一种“重量”

也就是说，反馈存在，但“权重排序”还不够明显。

### 2.4 UI 功能齐全，但实验感和战场感还可以更强

当前 UI 已经具备：

- 建造面板
- 顶部 HUD
- 控制栏
- Tooltip
- 胜利弹窗

问题不在功能缺失，而在于：

- 信息层级还不够强
- 面板仍偏“普通网页面板”
- 尚未形成“战术沙盘 / 战场控制台”的统一氛围

### 2.5 缩放与高密度战斗下的表现策略还不够明确

项目本身支持大地图与较高单位密度，因此未来会越来越需要：

- 远景简化
- 近景强化
- 特效预算分级
- 信息按重要性显示

否则随着单位和军营增多，画面容易进入“都在动，但看不出重点”的状态。

## 3. 本轮优化目标

本轮 UI 与特效优化应达成以下 6 个目标。

### 3.1 让玩家一眼能分辨“军营类别”和“战场角色”

玩家不需要打开 tooltip 才知道谁是：

- 前排
- 远程持续输出
- 点杀单位
- 范围单位
- 支援单位
- 攻城单位

### 3.2 让每种攻击方式都拥有独立体感

例如：

- 剑兵是轻快挥砍
- 盾兵是厚重顶击
- 弓兵是拉弓蓄势
- 投矛是明显重投
- 爆破是抛投并炸开
- 火炮是有后坐力的重炮
- 医疗是温和支援型抛射

### 3.3 让战场反馈有明确层级

需要让玩家明显感受到：

- 轻反馈：普通受击
- 中反馈：盾挡、治疗、投矛命中
- 重反馈：炸弹爆炸、火炮爆炸、营地摧毁、胜负结算

### 3.4 让 UI 更像“战场实验控制台”，而不是普通网页按钮集合

不要求复杂，但需要有更明确的世界观气质。

### 3.5 让缩放和密度变化下的战场始终有重点

近景看动作，远景看结构。

### 3.6 在保证性能的前提下，提高视觉记忆点

不能靠无限叠加特效来获得热闹感，而应依靠：

- 正确的层次
- 清晰的造型
- 稳定的视觉语言

## 4. 设计原则

### 4.1 优先强化可读性，而不是先追求华丽

如果一个特效看起来很热闹，但会遮挡战场判断，就不应优先。

### 4.2 优先放大差异，而不是平均美化所有元素

当前最重要的是把兵种、军营、武器、反馈差异拉开，而不是把所有元素同时做得更精细。

### 4.3 统一“玩具战场”视觉语言

建议统一到以下语汇：

- 积木
- 木头
- 布旗
- 箭羽
- 纸片
- 烟圈
- 小火花
- 彩色碎块

尽量减少与整体气质不一致的表现方式。

### 4.4 近景强调动作，远景强调结构

近景玩家看的是：

- 出手动作
- 命中特效
- 投射物飞行

远景玩家看的是：

- 哪边推进
- 哪类营地在输出
- 哪个区域正在爆炸

### 4.5 特效必须有优先级

不是所有特效都值得保真。

高优先级：

- 营地摧毁
- 火炮爆炸
- 胜负反馈
- 关键支援动作

低优先级：

- 普通多次小 hit
- 远景下的小粒子

## 5. 优化范围总览

本轮优化分为 6 个模块：

1. UI 风格与信息层级
2. 军营可视化强化
3. 单位与武器辨识度强化
4. 投射物与飞行表现优化
5. 命中 / 受击 / 爆炸特效重分层
6. 缩放与高密度战斗下的显示策略

---

## 6. UI 风格与信息层级优化

## 6.1 当前问题

当前 UI 的结构完整，但整体样式偏“功能型”：

- 字体偏默认
- 面板样式偏通用
- 颜色表达主要依赖红蓝区分
- 各区块之间的气质尚不够统一

### 6.2 优化目标

让整个 UI 更像：

- 战场沙盘控制台
- 玩具军团实验台

而不是普通网页控制条。

### 6.3 具体优化建议

#### 建造面板

目标：

- 更明确区分左右阵营
- 更强调军营身份
- 让玩家快速理解“这不是一排按钮，而是军营目录”

建议：

- 为每个军营按钮增加一句话定位
- 增加层级标签：基础 / 战术 / 特殊
- 当前选中态不只靠黄描边，可加入更强的面板发光或插旗感
- 锁定态不只靠锁图标，可加入“未解锁的暗面板感”

#### 顶部 HUD

目标：

- 第一眼就看出哪一方占优

建议：

- 强化“优势方”视觉状态
- 把单位存活、军营存活、击杀等信息分主次
- 用更强的布局节奏替代均匀排布
- 胜利前的优势趋势可以通过颜色或长度表达

#### Tooltip / 信息面板

目标：

- 不只显示数值，还显示用途

建议：

- 角色定位
- 优势标签
- 弱点标签
- 最佳用途

玩家应能从 tooltip 中快速回答：

- 这个营适合做什么
- 它怕什么

#### 控制栏

目标：

- 更清晰表达“模拟控制器”身份

建议：

- 播放 / 暂停 / 倍速更像一个时间控制模块
- 清空 / 重置 / 清兵等操作应与时间控制视觉分组分离

### 6.4 非目标

- 本轮不重做整个 UI 框架
- 不引入新前端库
- 不做复杂动画面板系统

---

## 7. 军营可视化强化

## 7.1 当前问题

军营外观已经有较清晰差异，但仍有提升空间：

- 血量低时缺乏更连续的破损阶段感
- 不同军营的“建筑身份”还可以更夸张
- 摧毁后残骸仍偏通用

### 7.2 优化目标

让玩家在远景下也能快速感知：

- 哪类军营在这一区域
- 哪些军营正在被重点攻击
- 哪些军营已经濒危

### 7.3 具体建议

#### 低血量阶段感

建议增加军营受损阶段表现：

- 轻伤：轻微暗化 / 少量裂缝
- 中伤：局部破损 / 倾斜小旗 / 边角碎块
- 重伤：明显裂纹 / 烟点 / 结构倾斜

#### 营地摧毁差异化

建议每种军营摧毁后保留一点身份痕迹，例如：

- 剑兵营：断裂塔垛、掉落剑饰
- 盾兵营：残盾、坍塌圆垒
- 弓兵营：断弓、散落木台
- 投矛营：断叉、倾斜尖桩
- 爆破营：炸裂火药箱残片
- 医疗营：倒下旗帜、碎裂十字标识
- 火炮营：熄火炮口、焦痕底座

#### 营地受击反馈

建议增加：

- 轻微局部抖动
- 掉落小积木块
- 瞬时灰尘或碎片

### 7.4 设计收益

- 营地不再只是“会掉血的建筑”
- 战场结构信息更容易读取

## 8. 士兵与武器辨识度优化

## 8.1 当前问题

当前单位造型已经比最早的占位图明显进步，但在中高密度战斗中，玩家仍然较难快速识别兵种。

### 8.2 优化目标

在不显著增加模型复杂度的前提下，让玩家远看也能分辨：

- 近战线
- 重装线
- 弓兵线
- 投矛线
- 爆破线
- 医疗线
- 火炮兵

### 8.3 具体建议

#### 轮廓优先

兵种区分应优先依赖轮廓，而不是细节。

建议：

- 剑兵：更轻、更前倾、更短促
- 盾兵：更宽、更低重心、盾面更大
- 弓兵：弓轮廓更明显
- 投矛兵：长矛更长、更高举
- 爆破兵：明显抱持炸药箱或桶
- 医疗兵：大十字包、旗帜或背箱更突出
- 火炮兵：持炮弹或短炮形态更明显

#### 战场缩放下的识别辅助

建议在中远景下启用轻量识别符号，例如：

- 头顶兵种色块
- 背旗
- 肩章或帽型差异

要求：

- 不能太吵
- 不能压过阵营色

#### 武器身份强化

武器不仅是装饰，应当是兵种身份核心的一部分。

建议强化：

- 弓兵的拉弓感
- 投矛兵的投掷感
- 爆破兵的重物抛投感
- 医疗兵的支援器具感
- 火炮兵的重炮装填 / 发射感

---

## 9. 投射物与飞行表现优化

## 9.1 当前问题

当前投射物已经有基础抛物线和影子，方向正确，但还缺少更强的“飞行身份”。

### 9.2 优化目标

玩家仅凭投射物，也能大致判断：

- 这是弓箭
- 这是投矛
- 这是爆破弹
- 这是治疗弹
- 这是火炮弹

### 9.3 具体建议

#### 弓箭

建议增加：

- 更轻的尾羽方向感
- 更短促的风痕
- 命中前更干净、利落

关键词：

- 轻
- 快
- 精准

#### 投矛

建议增加：

- 更明显的翻转和重量感
- 命中前更强的下坠感

关键词：

- 重投
- 穿刺

#### 爆破弹

建议增加：

- 更明显的火星尾迹
- 更夸张的抛物线重量感

关键词：

- 抛投
- 危险
- 近中程爆裂

#### 治疗弹

建议增加：

- 更柔和的脉冲光
- 更圆润的轨迹感
- 支援而非攻击的视觉语气

关键词：

- 温和
- 治愈
- 非侵略性

#### 火炮弹

建议增加：

- 烟尾
- 火焰余辉
- 更强的下坠压迫感

关键词：

- 沉重
- 远距
- 攻城

---

## 10. 命中、受击与爆炸特效重分层

## 10.1 当前问题

当前特效已经很多，但层级还不够清晰。

### 10.2 目标

将战斗反馈明确分为三档。

### 10.3 第一档：轻反馈

适用：

- 普通近战命中
- 普通箭命中
- 小单位受击

表现要求：

- 快
- 短
- 不挡视野

视觉建议：

- 小火花
- 短促闪白
- 微小抖动

### 10.4 第二档：中反馈

适用：

- 盾挡
- 治疗命中
- 投矛命中
- 关键单位被点杀

表现要求：

- 能看出身份
- 能区分于普通攻击

视觉建议：

- 盾挡：金属火花 + 圆弧反弹感
- 治疗：柔和十字 / 脉冲圈
- 投矛：更重的穿刺感 / 下沉感

### 10.5 第三档：重反馈

适用：

- 炸弹爆炸
- 火炮爆炸
- 营地摧毁
- 胜利结算

表现要求：

- 是战场焦点
- 权重明确高于普通命中

### 10.6 爆破营与火炮营的爆炸区分

这是本轮特效里最重要的一条。

#### 爆破营爆炸

关键词：

- 近中程
- 炸开
- 清人堆
- 快速散裂

建议：

- 更偏圆形爆散
- 更多火星、碎块、小烟环
- 节奏更短

#### 火炮营爆炸

关键词：

- 重击地面
- 冲击波
- 余波
- 焦痕

建议：

- 更强的地面冲击感
- 更重的烟波与震屏
- 更明显的落点焦痕
- 比爆破营更“厚重”、更“钝”

### 10.7 营地摧毁

建议强化为全局高优先级事件：

- 震动
- 结构坍塌
- 碎块飞散
- 烟圈
- 废墟残留

### 10.8 特效预算策略

本轮 PRD 要求特效系统分优先级处理：

- 高优先级特效尽量保留
- 低优先级小 hit 特效在高压场景可抽样或降级

不要求玩家明确察觉“特效被删了”，只要求：

- 重要信息不丢
- 画面不糊

## 11. 缩放与高密度战斗下的显示策略

## 11.1 当前问题

随着单位变多、画面拉远，视觉信息会急剧堆积。

### 11.2 优化目标

实现“近景看动作，远景看结构”的显示逻辑。

### 11.3 建议策略

#### 近景

重点显示：

- 单位动作
- 武器差异
- 命中反馈
- 单位血条

#### 中景

重点显示：

- 单位类型分组
- 军营输出方向
- 主要爆炸和命中区域

适度简化：

- 小粒子
- 普通受击细节

#### 远景

重点显示：

- 军营
- 战线推进
- 大爆炸
- 营地摧毁
- 优势方趋势

建议隐藏或弱化：

- 普通单位血条
- 小 hit 特效
- 细碎武器动作

---

## 12. 移动端与触控体验

虽然当前主要讨论 UI 和特效，但移动端需要同步考虑：

- 面板不能遮挡太多战场
- 特效不能过密
- 关键信息要更集中

建议：

- 移动端优先显示战局重点，不强调细碎装饰
- 胜利、营地摧毁、火炮爆炸等高价值反馈要保留
- 普通命中粒子可以适当减配

## 13. 本轮不做的内容

为了控制范围，本轮不包含：

- 大规模换美术资源
- Spine / 骨骼动画
- 复杂 shader
- 完整天气、昼夜、环境光系统
- 复杂地形装饰系统
- 大量新的音效系统设计

这些都可以作为后续版本方向，但不是本轮重点。

## 14. 推荐实施优先级

如果本轮只做最重要的三项，优先级建议如下：

### P1：统一视觉语言 + 拉开 UI 风格层级

原因：

- 这是整体气质的底层问题
- 能最快提升“这款游戏有自己的视觉身份”的感受

### P2：强化单位 / 武器 / 投射物辨识度

原因：

- 这直接影响战斗是否看得懂
- 比单纯做更多特效更值

### P3：重做特效层级，尤其区分爆破与火炮

原因：

- 这是当前最容易让战斗显得“同质”的地方
- 做好后，战场记忆点会明显提升

第二梯队：

- 军营破损阶段感
- 缩放 LOD 与特效预算细化
- HUD 优势表达强化

## 15. 验收标准

当以下条件成立时，可认为本轮 UI 与特效优化达标：

1. 玩家不打开 tooltip，也能大致看出各类军营与兵种职责
2. 爆破营与火炮营的战斗反馈能被明显区分
3. 医疗、盾挡、投矛等关键行为拥有独立识别感
4. 大规模战斗中仍能快速判断战线、优势方和关键威胁
5. UI 不再只是功能面板，而具备更统一的“战场实验台”气质
6. 特效热闹但不糊，重要信息不被淹没

## 16. 总结

当前版本已经有了一套能工作的 UI 和战斗表现系统，接下来最值得做的不是“加更多东西”，而是“让已有东西更有层次、更有身份、更有世界观一致性”。

这轮优化的核心不是纯美化，而是三件事：

1. 看懂
2. 记住
3. 不乱

只要这三点达成，这个项目的战斗观赏性和实验乐趣都会明显提升。
